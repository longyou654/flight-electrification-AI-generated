1
数据准备 — 用 pandas 玩转时间序列
1 – 2 天pandas
为什么要从这里开始？
不管后面用什么模型，80% 的时间都在处理数据。时间索引、重采样、滞后特征——这三样是基础功，基础不牢后面寸步难行。
核心操作

时间索引 .set_index() / pd.to_datetime()

让 DataFrame 知道哪一列是时间，才能做后续所有操作。

重采样 .resample()

把秒级数据聚合成分钟/小时/天，或者把日数据降成周/月。业务里最常用的操作之一。

滞后特征 .shift()

把昨天的值、上周的值当作特征。这是把时序问题转换成回归问题的关键一步。

滑动窗口 .rolling()

过去 N 天的均值、标准差，用来捕捉短期趋势和波动。

差分 .diff()

当前值减去上一个值。把非平稳序列变成平稳序列的核心手段。

# 五分钟上手示例
import pandas as pd

df = pd.read_csv('data.csv')
df['date'] = pd.to_datetime(df['date'])
df = df.set_index('date').sort_index()

df['lag_1'] = df['value'].shift(1)        # 昨天的值
df['ma_7']  = df['value'].rolling(7).mean()  # 7日均值
df['d1']    = df['value'].diff(1)           # 一阶差分
weekly  = df.resample('W').mean()            # 按周聚合
▶ 新手常踩的坑
  • 忘记 .sort_index()，导致 .shift() 结果错位
  • 把未来的数据当特征用了（look-ahead bias），这在实战里极其致命
  • 时区没统一，跨时区数据混着用导致时间戳对不齐
▶ 练手任务

找一个带日期的 CSV 数据集，完成：① 设为时间索引并排序 ② 画出原始值、7日均值、一阶差分三条线 ③ 生成 lag_1 ~ lag_7 共 7 个滞后特征。做完这三步，pandas 时序操作就算入门了。

2
经典方法 — ARIMA / ETS / Prophet
2 – 3 天statsmodels · prophet
为什么要先学经典方法？
这些模型可解释性强、部署轻量，在数据量不大的场景下效果不输深度学习。更重要的是——它们帮你建立对时序结构的直觉（趋势、季节、残差），这个直觉是你后面所有建模的基础。
三个模型

ARIMA / SARIMA

最经典的时间序列模型。AR（自回归）用过去的值预测现在，MA（移动平均）用过去的误差修正现在，I（差分）让序列变平稳，S（季节性）处理周期模式。
核心技能：学会看 ACF / PACF 图 来给模型定阶（p, d, q）。

Holt-Winters 指数平滑 (ETS)

思路更直观：预测值 = 水平 + 趋势 + 季节，每个成分做加权平滑，越近的数据权重越大。适合有明显季节性且趋势变化不剧烈的场景。

Prophet（Meta 出品）

拆成趋势 + 季节 + 节假日效应，自动检测变点。几行代码出结果、自带不确定性区间、对缺失值和异常值不敏感。最适合快速出业务 baseline。

▶ 新手常踩的坑
  • 不检查平稳性就直接套 ARIMA——先做 ADF 检验，不平稳就差分
  • 把 ACF/PACF 图当装饰不看，随手拍一个 (p,d,q) 就开始调参
  • Prophet 默认假设数据有强周期性，用在无周期数据上效果会很差
▶ 练手任务

用 Kaggle 的 Air Passengers 数据集：① 画时序分解图（trend / seasonal / residual）② 用 ADF 检验判断平稳性 ③ 看 ACF/PACF 图手动定一次阶 ④ 分别用 ARIMA 和 Prophet 做 12 步预测，对比 RMSE。

3
机器学习方法 — 特征工程 + 树模型
2 – 3 天XGBoost · LightGBM
为什么这一步最关键？
把时间序列转成「监督学习」问题后，你可以把天气、节假日、促销活动、竞品价格——任何外部变量都塞进去。这比 ARIMA 灵活得多，也是工业界最主流的做法。
核心思路

把时序变成表格

目标列 = 未来的值（比如明天），特征列 = 过去的 lag 值 + 滚动统计量 + 时间元信息（星期几、第几月、是否节假日）。每行是一个「用过去预测未来」的样本。

特征工程三件套

① 滞后特征：lag_1, lag_7, lag_30 — 不同时间尺度 ② 滚动统计：rolling mean/std/min/max of 7/14/30 days ③ 日期特征：dayofweek, month, quarter, is_weekend, is_holiday

XGBoost / LightGBM

把上面造好的特征喂给树模型做回归。天然处理缺失值、自动捕捉非线性关系、特征重要性能看出哪些因素最关键。大部分时间序列比赛和工业项目的标配。

▶ 新手常踩的坑
  • 数据泄露！用未来的信息造特征，比如用本周的均值去预测昨天的值
  • 切训练/测试集时随机 shuffle——时间序列必须按时间顺序切分
  • 只用了 lag 特征忘了外部特征（天气、促销、节假日），浪费了树模型最大的优势
▶ 练手任务

用同样的 Air Passengers 数据集：① 构造 lag_1 ~ lag_12 + rolling_mean_6/12 + month 特征 ② 用 LightGBM 做 12 步预测 ③ 对比和 ARIMA 的误差 ④ 画出 feature importance，看哪些特征贡献最大。

4
深度学习方法 — 选学
按需PyTorch · pytorch-forecasting
什么时候才需要深度学习？
数据量极大（几十万条以上）、序列之间相互关联（比如几千个产品的销量一起预测）、或者前三个阶段的方法都试过了确实不够用。大部分业务场景前三个阶段就够了。

LSTM / GRU

循环神经网络的变体，能记住长距离的依赖关系。但调参麻烦、训练慢、容易过拟合——别一上来就用。

N-BEATS / TFT

更现代的专用时序架构。N-BEATS 纯 MLP 结构简单效果好；TFT（Temporal Fusion Transformer）能同时处理静态特征、已知未来（比如假期日期）和未知未来。都在 pytorch-forecasting 里有现成实现。

▶ 新手常踩的坑
  • 数据才几千条就上 LSTM——模型容量远大于数据量，必然过拟合
  • 不做 baseline 直接上深度学习，最后发现比 ARIMA 还差
  • 忘记做数据归一化，LSTM 对输入尺度敏感
